{\footnotesize\ttfamily \mbox{\hyperlink{namespacegason_af84c9b75c32aa947dc35e42d003cca0b}{Json\+Parse\+Status}} gason\+::json\+Parse (\begin{DoxyParamCaption}\item[{char $\ast$}]{json\+String,  }\item[{\mbox{\hyperlink{structgason_1_1JsonValue}{Json\+Value}} \&}]{root,  }\item[{\mbox{\hyperlink{classgason_1_1JsonAllocator}{Json\+Allocator}} \&}]{allocator }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}}

parses a JSon string and return root object or array. Json\+Values are the index tree for actual data on json\+String. allocator object holds and manage these indices.

to keep the \mbox{\hyperlink{structgason_1_1JsonValue}{Json\+Value}} valid, you have to keep both json\+String (data) and allocator (instances) alive.

to avoid calling alloc/free many times (a serious problem on embedded devices) the allocator is re-\/usable and you can hold and re-\/use instance for parsing as many as json\+Strings you want. \mbox{\hyperlink{classgason_1_1JsonAllocator}{Json\+Allocator}} automatically expands if more memory is required and frees all memories automatically.


\begin{DoxyParams}{Parameters}
{\em json\+String} & the JSon string. this buffer will be modified by this function. \\
\hline
{\em root} & root object of JSon string. could be an object or array. \\
\hline
{\em allocator} & an instance to memory manager. this function calls allocator.\+reset() before parsing. \\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
returns JSON\+\_\+\+PARSE\+\_\+\+OK or a proper error code. 
\end{DoxyReturn}
